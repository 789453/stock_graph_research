# AI Working Protocol

## 1. 这不是“请你自由发挥”的项目

AI 在本项目中的任务，是在既定研究边界内完成确定动作。  
当你不确定时，应当：

1. 回看文档
2. 回看真实数据契约
3. 报告冲突
4. 停止扩大范围

而不是继续堆代码。

## 2. 每轮固定执行顺序

1. 读文档
2. 复述本轮任务边界
3. 列出将修改的文件
4. 修改代码
5. 运行最小真实测试
6. 检查缓存产物
7. 更新 `PROJECT_STATE.md`
8. 给出本轮差异报告

## 3. 禁止行为

- 禁止为了跑通而改研究目标
- 禁止偷偷使用 mock / sample / fallback
- 禁止从“构图”顺手扩展到“聚类、平滑、因子、回测”
- 禁止只给一段解释，不留下文件
- 禁止把描述性结果说成工程成功
- 禁止忽略缓存，反复重算上游
- 禁止删除失败证据

## 4. 当你想加一个新抽象时，先问

- 这个抽象是否服务于当前任务？
- 它是否减少了当前轮的错误面？
- 它是否会让研究问题更清楚？
- 如果没有它，当前轮是否仍可完成？

若前三个答案不是“是”，通常不要加。

## 5. 当你想写测试时，先问

- 我定义的“正确”是什么？
- 这是不是工程契约，而不是研究结果？
- 这个测试是否依赖真实数据身份？
- 这个测试失败后，研究者能否更快知道哪里错了？

## 6. 当你发现文档和代码不一致时

按以下格式报告：

```text
发现冲突：
1. 文档规定：
2. 当前代码：
3. 影响：
4. 建议处理：
5. 本轮是否继续：
```

没有研究者确认前，不要把项目改造成另一种设计。

## 7. 当你完成一个任务时，必须给出

```text
本轮完成：
- 任务：
- 修改文件：
- 真实输入：
- 缓存输出：
- 测试：
- 已知限制：
- 下一步唯一建议：
```

## 8. 最重要的一句话

> 工具不能解决目的模糊的问题，  
> 模型不能解决规格不清晰的问题。  
> 你的职责不是替研究者跳过思考，而是让已经想清楚的问题被可靠执行。
